# multi AI

## 概要・説明

**multi AI** は、1つのアプリケーションから複数の生成AIプロバイダー（Google Gemini, Anthropic Claude, OpenAI GPT, xAI Grok）をシームレスに利用できる、高機能なシングルページ・チャットアプリケーションです。

「Obsidian Terminal」風の洗練されたダークテーマを採用し、レスポンシブデザインによりPC・モバイルの両方で快適に動作します。バックエンドサーバーを必要とせず、ブラウザのローカルストレージ（IndexedDB）とWeb Workerを活用することで、セキュアかつ高速な動作を実現しています。

画像やテキストファイルの読み込み（マルチモーダル対応）や、ローカルフォルダへの履歴の自動保存機能など、AIを日常的・本格的に活用するユーザー向けの高度な機能が詰め込まれています。

---

## 主な機能一覧

* **マルチプロバイダー対応**
  * Gemini, Claude, GPT, Grok の主要な生成AIモデルをワンクリックで切り替え可能。
  * ユーザー自身で新しいカスタムモデル名を追加・編集・削除可能。
* **マルチモーダル＆ファイル添付**
  * 画像ファイル（JPEG, PNG, GIF, WebP）を添付し、画像認識AIへ送信。
  * テキストファイルやMarkdownファイルを添付し、プロンプトのコンテキストとして読み込み。
* **柔軟なシステムプロンプト管理**
  * 「アシスタント」「プログラマー」「翻訳者」など、あらかじめ設定した役割（システムプロンプト）を管理し、チャットごとに素早く切り替え。
* **ローカルファーストなデータ管理**
  * チャット履歴や設定はすべてブラウザのIndexedDBに保存され、外部サーバーには送信されません（APIリクエストを除く）。
  * 履歴の全文検索や、過去のチャットの再開が可能。
* **ファイルシステム連携（自動保存）**
  * File System Access APIを活用し、指定したローカルPCのフォルダにチャット履歴をJSON, Markdown, またはテキスト形式で自動保存。
* **バックアップと復元**
  * 全データのJSONエクスポート・インポート機能。
* **快適なUI/UX**
  * Web Workerを利用したバックグラウンド処理により、UIをブロックせずにAIのストリーミング応答をリアルタイム表示。
  * Markdownのレンダリング、シンタックスハイライト対応。
  * メッセージやコードブロックのワンクリックコピー機能。
  * 生成中の画面消灯を防ぐ Screen Wake Lock 対応。

## 使用技術・ライブラリ

本プロジェクトはビルドツールを必要とせず、単一のHTMLファイル（HTML/CSS/Vanilla JavaScript）で構成されています。

### フロントエンド
* **HTML5 / CSS3** (CSS Variablesによるデザインシステム、レスポンシブ対応)
* **Vanilla JavaScript** (ES6+)

### ブラウザAPI
* **IndexedDB**: ローカルデータベース
* **Web Worker API**: AI通信・ストリーミング処理の非同期化
* **File System Access API**: ローカルフォルダへの直接保存
* **Screen Wake Lock API**: 処理中のスリープ防止

### 外部ライブラリ (CDN経由)
* **[Dexie.js](https://dexie.org/)** (v3.2.4): IndexedDBのラッパーライブラリ
* **[Marked.js](https://marked.js.org/)** (v9.1.2): Markdownのパース・HTML変換
* **[DOMPurify](https://github.com/cure53/DOMPurify)** (v3.0.6): クロスサイトスクリプティング（XSS）対策・サニタイズ
* **Font Awesome** (v6.4.0): UIアイコン
* **Google Fonts**: Syne, DM Sans, JetBrains Mono

## セットアップ・使い方

### 1. セットアップ（インストール）
このアプリケーションはサーバー構築が不要です。

1. リポジトリまたは提供されたコードを `index.html` として任意のフォルダに保存します。
2. セキュリティ制約（Web WorkerやFile System Access APIの利用）を回避するため、ローカルサーバー経由で開くことを強く推奨します。
   * VS Code拡張機能の **Live Server** を使うか、ターミナルで `python -m http.server` または `npx serve` などを実行してください。
   * ブラウザで `http://localhost:XXXX` にアクセスします。

### 2. 初期設定（APIキーの登録）
1. 画面右上の **設定アイコン（歯車マーク）** をクリックします。
2. 「APIキー設定」のセクションに、利用したいプロバイダー（Gemini, Claude, OpenAI, Grok）のAPIキーを入力します。
3. **「APIキーを保存」** ボタンをクリックします。

### 3. 基本的な使い方
1. **チャットの開始**: 左側のサイドバー（モバイルの場合は左上のメニューボタン）から「新しいチャット」をクリックします。
2. **AI設定の変更**: メッセージ入力欄の左にある **ペンアイコン（AI設定）** をクリックし、使用する「役割（プロンプト）」「プロバイダー」「モデル」を選択します。
3. **メッセージの送信**: テキストエリアにメッセージを入力し、Enterキー（または右側の送信ボタン）でAIに送信します。
4. **ファイルの添付**: 入力欄左の **クリップアイコン** をクリックし、画像やテキストファイルを添付できます。プレビューが表示された状態で送信すると、AIにそのファイル情報が渡されます。

### 4. 履歴のローカル保存設定（任意）
1. 設定画面を開き、「ファイルシステム連携 (自動保存)」セクションに移動します。
2. 保存したい形式（Markdown, JSON, テキスト）を選択します。
3. **「保存フォルダを選択」** をクリックし、PC内の任意のフォルダへのアクセスを許可します。
4. 以降、チャットを行うたびに、指定したフォルダへチャット内容のファイルが自動で生成・更新されます。
