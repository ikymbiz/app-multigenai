# my AI

## 概要・説明
**my AI** は、ブラウザ単体で完結し、複数の主要な生成AI（LLM）プロバイダーと直接通信できる高機能なフロントエンド・チャットアプリケーションです。
サーバーやバックエンド環境を一切必要とせず、一つのHTMLファイル（内部にCSS/JSを含む）をブラウザで開くだけで動作します。

OpenAI (GPT)、Anthropic (Claude)、Google (Gemini)、xAI (Grok) に対応しており、APIキーを設定するだけで好みのAIモデルを自由に切り替えて会話できます。
モダンなダークテーマ（Obsidian Terminalスタイル）を採用し、Markdownレンダリング、ファイル添付機能、システムプロンプトのカスタマイズ、ローカル環境へのデータ自動保存など、日常的なAIアシスタントとして本格的に使える設計になっています。

## 主な機能一覧

### 🤖 マルチLLM・プロバイダー対応
- **4大プロバイダー対応**: Gemini, Claude, GPT, GrokのAPIに標準対応。
- **カスタムモデルの追加**: 標準のモデル（gpt-4o, claude-3-opus等）に加え、新しいモデル名を自由に登録・編集可能。
- **ストリーミング応答**: Web Workerを利用し、AIの回答をリアルタイムでスムーズに画面へ描画。

### 🎭 システムプロンプトと役割管理
- **プリセットの役割**: 「アシスタント」「プログラマー」「マーケター」など、用途に応じたシステムプロンプトを内蔵。
- **カスタムロール作成**: 自分専用のシステムプロンプトを持つ「役割」を新規追加・保存可能。

### 📎 高度なファイル添付機能
- **画像プレビュー・解析**: ドラッグ＆ドロップやファイル選択で画像を添付。Base64化してAIに送信可能（Vision対応モデルで画像認識が可能）。
- **テキスト・Markdownファイルの読み込み**: ファイル内のテキストデータを抽出してプロンプトのコンテキストとして追加。

### 💾 堅牢なデータ管理とプライバシー
- **ローカルデータベース**: IndexedDB (Dexie.js) を利用し、チャット履歴、設定、APIキーをブラウザ内に安全に保存。
- **File System Access API 連携**: 指定したローカルのフォルダに、チャット履歴をJSON, Markdown, またはテキスト形式で自動保存（バックアップ機能）。
- **インポート / エクスポート**: すべてのチャットデータや設定を手動でJSONファイルとして書き出し、別のブラウザで復元可能。

### 🎨 リッチなUI / UX
- コードのシンタックスハイライト、テーブル、画像の表示など、完全なMarkdownレンダリング。
- スマートフォンでも使いやすいレスポンシブデザイン（サイドバーの開閉など）。
- ワンクリックでAIのメッセージをクリップボードにコピー。
- 画面スリープ防止機能（Wake Lock API）を搭載し、長文生成中の画面消灯を防止。

## 使用技術・ライブラリ

本アプリケーションは、外部サーバーへの依存をなくすため、フロントエンド技術のみで構築されています。

- **言語**: HTML5, CSS3, Vanilla JavaScript
- **ストレージ・データ管理**: 
  - `IndexedDB` (ローカルデータベース)
  - `File System Access API` (ローカルファイルへの直接保存)
- **非同期処理**: `Web Worker` (メインスレッドをブロックしないAPI通信とストリーミング処理)
- **外部ライブラリ (CDN経由で読み込み)**:
  - [Dexie.js](https://dexie.org/) (v3.2.4) - 使いやすいIndexedDBラッパー
  - [Marked.js](https://marked.js.org/) (v9.1.2) - 高速なMarkdownパーサー
  - [DOMPurify](https://github.com/cure53/DOMPurify) (v3.0.6) - 安全なHTMLレンダリングのためのXSSサニタイザ
  - Font Awesome (v6.4.0) - UIアイコン
  - Google Fonts - タイポグラフィ (Syne, DM Sans, JetBrains Mono)

## セットアップ・使い方

### 1. 起動方法
サーバーの構築や依存パッケージのインストールは不要です。
ダウンロードしたHTMLファイルを、Google ChromeやMicrosoft Edgeなどのモダンブラウザでそのまま開いてください。

### 2. 初期設定（APIキーの登録）
1. 画面右上の **歯車アイコン（設定）** をクリックして設定画面を開きます。
2. 「APIキー設定」のセクションにて、利用したいAIプロバイダーのAPIキーを入力します。
   - *※ APIキーはブラウザのローカルストレージ(IndexedDB)に暗号化されずに保存されます。共有のPCでは使用しないでください。*
3. 「APIキーを保存」ボタンをクリックします。

### 3. チャットの開始
1. メッセージ入力欄の左側にある **ペンアイコン（AI設定）** をクリックします。
2. 対話する「役割」、使用する「プロバイダー」、そして「モデル」を選択します。
3. 入力欄にメッセージを入力し、右下の送信ボタン（またはEnterキー）で送信します。

### 4. ファイルの添付
- 入力欄左側の **クリップアイコン** をクリックし、画像（JPEG/PNG/WebPなど）やテキストファイルを選択します。
- 添付されたファイルはプロンプトのコンテキストとしてAIに送信されます。

### 5. チャット履歴の自動保存（任意）
1. 設定画面を開き、「ファイルシステム連携 (自動保存)」セクションに移動します。
2. 「保存形式」を選び、「保存フォルダを選択」をクリックします。
3. ブラウザのダイアログに従い、PC上の任意のフォルダに対して読み書き権限を許可します。
4. 以降、チャットを生成するたびに、指定したフォルダに会話ログが自動的にファイルとして保存されます。
